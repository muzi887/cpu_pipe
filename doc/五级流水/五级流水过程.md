# 五级流水过程文字说明

## 一、流水线是什么

五级流水线把一条指令的执行拆成 **5 个固定阶段**，每个时钟周期各阶段**同时工作**，但处理的是**不同指令**的不同步骤：

```text
时钟周期 n:   IF(i+4) | ID(i+3) | EX(i+2) | MEM(i+1) | WB(i)
```

和你原来的微程序模型机对比：原来一条指令要多个微周期串行占用单总线（`PC→MAR→RAM→MDR→IR→…`）；流水化后，5 个阶段**重叠执行**，理想情况下每拍完成一条指令，吞吐率接近 **1 IPC**（每周期一条）。

段与段之间用 **流水线寄存器** 隔开，在时钟上升沿锁存上一段的结果，供下一段使用。这是从“单总线分时复用”改为“专用数据通路 + 段间寄存器”的核心。

## 三、段间流水线寄存器（隔离各级的“缓冲器”）

| 寄存器 | 锁存内容 | 作用 |
|--------|----------|------|
| **IF/ID** | pc, pc_plus1, instr | 隔离 IF 与 ID；分支误取时 flush 为 NOP |
| **ID/EX** | pc, rs1_val, rs2_val, rs1, rs2, rd, imm, control | 隔离 ID 与 EX；load-use stall 时插入 bubble |
| **EX/MEM** | alu_result, rs2_val, rd, branch_taken, branch_target, control | 隔离 EX 与 MEM；供转发单元使用 |
| **MEM/WB** | mem_data, alu_result, rd, control | 隔离 MEM 与 WB；供转发单元使用 |

每个时钟上升沿，各级组合逻辑的结果锁入下一段寄存器，保证多条指令在流水线中**不会互相干扰**。

## 四、辅助器件（保证流水正确运行）

### control_unit（控制器）

- **功能**：在 ID 阶段根据 opcode 产生全部控制信号。
- **与原机区别**：不再使用 `microController`、CM_ROM、微指令序列；改为固定 5 级 + 控制信号随 ID/EX → EX/MEM → MEM/WB 向后传递。

### forwarding_unit（数据转发单元）

- **功能**：解决 **RAW 数据冒险**——后面指令在 EX 就要用前面指令的结果，但写回要到 WB 才完成。
- **做法**：若 EX/MEM 或 MEM/WB 的目标寄存器等于当前 EX 需要的 rs1/rs2，且会写回，则通过 Forward MUX 直接把 `alu_result` 或 `write_data` 送给 ALU，不必等 WB。
- **典型场景**：`ADD x3, x1, x2` 后紧跟 `ADDI x2, x3, 0`。

### hazard_unit（冒险处理单元）

- **功能**：检测并处理流水线冲突，发出 stall、flush 信号。
- **load-use stall**：`LD` 后下一条立刻使用 load 结果时，转发不够，需 **冻结 PC 和 IF/ID 1 拍**，向 ID/EX 插入 bubble。
- **分支 flush**：BNE 在 EX 才确定跳转，IF/ID 中已误取的 1–2 条指令作废，PC 改为 `branch_target`，IF/ID、ID/EX 清空为 NOP。

## 五、一条指令从头到尾怎么走（以 `ADD x3, x1, x2` 为例）

假设流水线已填满，该指令为指令 i：

| 周期 | 阶段 | 发生的事 |
|------|------|----------|
| 1 | **IF** | PC 指向 i，从存储器取出 i 的 machine code，锁入 IF/ID |
| 2 | **ID** | 译码为 ADD，读 x1、x2，产生 RegWrite/ALUOp 等，锁入 ID/EX |
| 3 | **EX** | ALU 计算 x1+x2，得到 alu_result，锁入 EX/MEM |
| 4 | **MEM** | 不访存，alu_result 直通，锁入 MEM/WB |
| 5 | **WB** | MemToReg 选 alu_result，写入 x3 |

5 拍后 x3 更新完成。与此同时，i+1、i+2… 也在各级并行推进。

## 六、和原微程序模型机的整体对比

| 维度 | 原多周期 CPU | 五级流水 CPU |
|------|--------------|--------------|
| 控制 | 微程序 ROM，多微周期 | 硬布线，固定 5 级 |
| 数据通路 | 单总线 + Bus_Sel 分时 | 段间专用线 + 转发 MUX |
| 寄存器 | PC/MAR/MDR/IR/AC/AX/BX/CX | PC + reg_file + 流水线寄存器 |
| 一条指令 | 十几个微周期串行 | 5 个时钟周期（可与其它指令重叠） |
| 分支 | PSW_Z + JNZ 微跳转 | EX 比较 + flush |
| 冒险 | 无（天然串行） | 转发 + stall + flush |

## 七、一句话记住每一级

- **IF**：按 PC 取指令  
- **ID**：看懂指令、读寄存器、产生控制信号  
- **EX**：ALU 算数、算地址、判分支  
- **MEM**：读/写数据存储器  
- **WB**：结果写回寄存器堆  

五级之间靠 **IF/ID、ID/EX、EX/MEM、MEM/WB** 四个流水线寄存器衔接；**control_unit** 负责译码，**forwarding_unit** 和 **hazard_unit** 负责保证在多条指令并行时结果仍然正确。
