这一步的目标是：**用 4～5 条互不复杂的指令，分别验证“写寄存器、做加法、写内存”三件事**。做法如下。

## 第一步：准备最小测试程序

文档里的 4 条指令里，`ST x3, 0(x4)` **依赖 x4 已有基地址**，所以实际要 5 条：

```asm
0: ADDI x1, x0, 1    ; x1 = 1
1: ADDI x2, x0, 2    ; x2 = 2
2: ADD  x3, x1, x2    ; x3 = 3
3: ST   x3, 0(x0)    ; Mem[0] = 3
4: HALT
```

如果坚持用 `ST x3, 0(x4)`，前面还要加一条：

```asm
ADDI x4, x0, 13      ; x4 = 13，然后 Mem[13] = 3
```

对应机器码（按你的 16 位编码）：

| 地址 | 汇编 | 机器码 |
|:----:|------|:------:|
| 0 | ADDI x1, x0, 1 | `0041` |
| 1 | ADDI x2, x0, 2 | `0082` |
| 2 | ADD x3, x1, x2 | `1298` |
| 3 | ST x3, 0(x0) | `30C0` |
| 4 | HALT | `F000` |

修改 `cpu_pipe/rtl/instr_memory.vhd` 的 `INIT_MEM`：

```vhdl
constant INIT_MEM : mem_array_t := (
  0  => x"0041",
  1  => x"0082",
  2  => x"1298",
  3  => x"30C0",
  4  => x"F000",
  others => (others => '0')
);
```

`data_memory.vhd` 可以保持全 0 初值，方便看 `ST` 是否写进去。

## 第四步：按“预期结果”对照

五级流水有延迟，**不是取指当拍就写回**，一般要等该指令流到 WB 才看到寄存器变化。

预期最终状态：

| 检查项 | 预期 |
|--------|------|
| `regs(1)` | `0001` |
| `regs(2)` | `0002` |
| `regs(3)` | `0003` |
| `memory(0)` | `0003`（若用 `ST x3, 0(x0)`） |
| `wb_en` | 前 3 条 ADDI/ADD 各脉冲一次 |
| `mem_write_en` | `ST` 到 MEM 时拉高 1 拍 |

对照时可以这样看：

1. `wb_addr = 1` 且 `wb_data = 1` → 第一条 ADDI 写回成功  
2. `wb_addr = 2` 且 `wb_data = 2` → 第二条 ADDI 写回成功  
3. `wb_addr = 3` 且 `wb_data = 3` → ADD 写回成功  
4. `mem_write_en = 1` 且 `mem_addr = 0`、`mem_wdata = 3` → ST 成功  

## 第五步：判断“通过 / 不通过”

**通过**：上述 4 个现象都能看到，且数值正确。

**不通过时优先查**：

| 现象 | 可能原因 |
|------|----------|
| `x1/x2` 对，`x3` 错 | `ADD` 的 rs2 读错，或 forwarding 有问题 |
| 寄存器都对，内存没写上 | `ST` 的 S 型字段解析错，或 `mem_write_en` 没到 MEM |
| 全部不对 | `id_stage` 控制信号或写回使能有问题 |
| `x3` 变成 1 或 2 | `ADD` 执行时 `x2` 还没写回，需要 forwarding |

注意：`ADD x3, x1, x2` 紧跟两条 `ADDI` 时，**很可能需要数据转发**。如果 `x3` 不对，先检查 `ex_stage.vhd` 里 `forward_b` 默认分支是否选 `alu_b_src`，而不是 `operand_b` 自己。
