# 中期任务计划 · 五级流水 CPU 核心闭环

> **依据**：最终稿数据流图（IF/ID/EX/MEM/WB + 寄存器堆 + 硬布线控制）  
> **课设题目**：基于 VHDL 的五级流水 RISC CPU 设计与存储/中断扩展——以斐波那契程序为验证负载  
> **中期目标**：完成**不含 Cache、不含完整冒险处理**的五级流水最小闭环，斐波那契程序结果正确  
> **RTL 目录**：`rtl_pipe/`  
> **参考文档**：`doc/0/流水线CPU-Cache-中断-嵌入式课设-初期报告.md`

---

## 1. 中期范围说明

### 1.1 本阶段要实现（对应数据流图）

| 数据流图模块 | 中期实现方式 | 说明 |
|--------------|--------------|------|
| **PC + PC_en** | `pc_reg.vhd` + Next-PC MUX | 仅支持 PC+1 顺序取指 |
| **i_cache** | **暂用 `instr_mem` 直连主存** | 单周期取指，接口预留 `i_miss` |
| **IF/ID**（`IF_ID_en`） | `pipe_if_id.vhd` | 锁存 `pc, pc+1, instr` |
| **Control Unit** | `control_unit.vhd` | 硬布线译码，替代微程序控制器 |
| **Registers** | `reg_file.vhd` | 双读单写，x0 恒 0 |
| **Unsigned Extend** | ID 阶段组合逻辑 | 6 位 imm/offset 符号扩展 |
| **ID/EX** | `pipe_id_ex.vhd` | 锁存 rs/rd 值、寄存器号、控制信号 |
| **ALU** | `alu.vhd` | ADD（斐波那契够用） |
| **EX/MEM** | `pipe_ex_mem.vhd` | 锁存 alu_result、写数据、rd |
| **d_cache** | **暂用 `main_memory` 直连** | 单周期访存，接口预留 `d_miss` |
| **MEM/WB** | `pipe_mem_wb.vhd` | 锁存 mem_data / alu_result |
| **WB MUX**（MemToReg） | WB 阶段 MUX | 选 ALU 结果或 load 数据写回 |

### 1.2 本阶段暂不实现（留最终阶段）

| 模块 | 原因 |
|------|------|
| **Hazard Detect Unit** | 中期用「无冒险」或手工插 NOP 的测试程序验证 |
| **Forwarding Unit** | 同上 |
| **Cache_control** | 无 i_cache / d_cache 实装 |
| **Stack for call/ret** | 斐波那契不需要 CALL/RET |
| **Flags（czvs）** | BNE 在 EX 阶段直接比较 rs1≠rs2 即可 |
| **Timer 中断 / MMIO** | 最终阶段 SoC 扩展 |

### 1.3 中期验收标准

| 编号 | 验收项 | 预期结果 |
|------|--------|----------|
| M1 | 五级流水并行 | 波形中可见不同指令处于 IF/ID/EX/MEM/WB |
| M2 | 斐波那契正确性 | `x3 = 13`，`Mem[13..17] = 2, 3, 5, 8, 13` |
| M3 | 控制信号流水传递 | RegWrite/MemRead 等于 ID 产生、WB 生效 |
| M4 | 写回目标正确 | `Wr_addr` 来自 MEM/WB.rd，非当前 IF/ID 指令 |
| M5 | 单周期取指/访存 | IF 连 instr_mem，MEM 连 main_memory，无 miss stall |

## 2. 数据通路对照（中期简化版）

```text
                    ┌─────────────────────────────────────────────────────────┐
  PC ──► instr_mem ──► IF/ID ──► Control + reg_file + imm_ext ──► ID/EX     │
    ▲      (代替           │              │                              │    │
    │       i_cache)      │              └── control 锁入 ──────────────┘    │
    │                     │                                                   │
  PC+1 MUX                │         EX: ALU (无 Forward MUX)                 │
  (仅 +1)                 │              │                                   │
                          │              ▼                                   │
                          │         EX/MEM ──► main_memory ──► MEM/WB      │
                          │              (代替 d_cache)          │           │
                          │                                      ▼           │
                          └──────────────────────────── reg_file 写回 ◄───────┘
```

> 最终稿完整数据流图见项目附件；中期在 IF/MEM 两级用主存占位，段间寄存器与控制通路结构与终稿一致。

| 序号 | 任务 | 产出文件 | 验收 |

| A1 | 16×16 寄存器堆：双读单写，x0 只读为 0 | `reg_file.vhd` | testbench 写 x1 读回正确 |
| A2 | ALU：ADD，输出 Zero 标志 | `alu.vhd` | 3+5=8，Zero 正确 |

### 阶段 C：PC 与分支（第 5 天）

| 序号 | 任务 | 说明 | 验收 |
|------|------|------|------|
| C1 | Next-PC MUX（2 选 1） | 0=PC+1，1=branch_target | BNE 成立时 PC 跳转 |
| C2 | 分支目标计算 | branch_target = ID/EX.pc + sign_ext(offset) | offset=−6 时回到 loop |
| C3 | 分支 flush（简化） | BNE 在 EX 成立：IF/ID←NOP，ID/EX←NOP | 误取 2 条指令被清空 |
| C4 | HALT 处理 | PC 自环或拉低 running | 仿真停止 |

> **说明**：终稿 Hazard Detect 在 ID 级处理 J/条件跳转；中期 BNE 仍在 **EX 级**判断并 flush，与初期报告 6.3 节一致，降低中期难度。

## 5. 中期测试策略

### 5.1 分步仿真（由简到繁）

| 步骤 | 测试程序 | 目的 |
|------|----------|------|
| T1 | 单条 ADDI | 验证 reg_file + WB |
| T2 | ADD + ADDI 写回链 | 验证 ID/EX→EX/MEM→MEM/WB |
| T3 | ST + LD | 验证 MEM 阶段 |
| T4 | 3 条指令流水重叠 | 验证 IF/ID/EX/MEM/WB 同时有指令 |
| T5 | 完整斐波那契（无 RAW 冲突版） | 先用手工调度避免数据相关 |
| T6 | 完整斐波那契（原始汇编） | 预期部分 RAW 结果错误 → 推动最终阶段做转发 |

### 5.2 中期已知问题（预期）

| 场景 | 现象 | 最终阶段解决方式 |
|------|------|------------------|
| ADD 后立刻 ADDI 用其结果 | 读到旧寄存器值 | Forwarding Unit |
| LD 后立刻使用 load 数据 | 结果错误 | Hazard Detect → load-use stall |
| BNE 跳转 | 已做 flush，但无 czvs 条件跳转 | Hazard Detect + Flags |
| 连续访存 | 无 miss penalty | Cache_control + d_cache |
