## 推荐题目（按优先级）

### ⭐ 第一梯队：强烈推荐（延续现有 CPU，课程覆盖最好）

#### 1. 五级流水线 CPU + 数据冒险/控制冒险处理
**做什么：** 在现有 CPU 上改为 IF→ID→EX→MEM→WB 五级流水，加 Forwarding、流水线停顿（stall）、分支预测或 flush。

**验证任务：** 斐波那契 / 冒泡排序 / 矩阵点积（比斐波那契更能暴露 RAW 冒险）。

**课程对应：** Ch5 流水线时空图、吞吐率/效率/加速比、 hazard 分析。

**优点：** 和组成原理、Ch5 完全衔接；答辩时能画时空图、算性能指标；就业面试也常见。

**工作量：** 中等（2–4 周），比从零做向量机现实。

---

#### 2. 带 Cache 的 CPU（L1 直接映射或 2 路组相联）
**做什么：** 在 CPU 与主存之间加 I-Cache / D-Cache，实现 tag/index/offset 译码、命中/缺失、写回或写直达；统计 hit rate、平均访存周期。

**验证任务：** 同一程序在「无 Cache vs 有 Cache」下对比周期数；或用不同局部性程序（顺序访问 vs 随机访问）对比命中率。

**课程对应：** Ch3 多体交叉存取、存储层次；Ch1 透明性（Cache 对软件是否透明）。

**优点：** 你已有主存接口，扩展自然；报告里能写 Amdahl 式分析（访存瓶颈占比）。

**工作量：** 中等；若只做直接映射 D-Cache，可控。

---

#### 3. 标量 CPU → 简易向量处理单元（SIMD 风格）
**做什么：** 不是完整 CRAY-1，而是在现有 ALU 旁加**向量寄存器组**（如 VR0–VR3，长度 4/8），新增向量加/乘/load/store 指令；可选「链接」：前一条向量指令结果直接作为下一条输入（简化版 Chaining）。

**验证任务：** 两个 4 元素数组相加、点积、斐波那契批量生成。

**课程对应：** Ch6 向量指令、链接、向量 vs 标量加速比。

**优点：** 对应你提到的「向量处理机」；比完整阵列机简单很多。

**工作量：** 中偏大；建议向量长度固定为 4 或 8，不要一开始就做通用向量机。

---

### ⭐ 第二梯队：有特色，适合想差异化

#### 4. 多周期 CPU + 中断与简单 I/O
**做什么：** 保留现有多周期结构，加中断控制器：定时器中断、键盘中断（仿真）；实现中断屏蔽位、优先级、现场保存/恢复。

**验证任务：** 斐波那契计算过程中响应定时器，或按键改变 n。

**课程对应：** Ch3 中断屏蔽（作业题 5、6 类大题）；软硬件分工。

**优点：** 和现有设计兼容最好；答辩可画「程序运行 + 中断插入」时序图。

**工作量：** 中等；控制逻辑比 Cache 复杂，但数据通路改动小。

---

#### 5. 嵌入式 SoC：CPU + 外设（UART/GPIO/Timer）
**做什么：** CPU 作为核心，通过内存映射 I/O 控制 LED、七段数码管、简单 UART；程序在「裸机」上跑斐波那契或计数器。

**验证任务：** 斐波那契结果通过数码管/串口输出；按键设定 n。

**课程对应：** Ch3 I/O 与通道（可对比程序查询 vs 中断驱动）；Ch1 系列机/软硬件接口。

**优点：** 对应「结合嵌入式」；展示完整、答辩直观。

**工作量：** 中等；若用 FPGA 板子加分，纯仿真也可。

---

#### 6. 双发射超标量或 2 路超流水线（简化版）
**做什么：** 2 条并行流水级，或同一周期发射 2 条无相关指令；画超标量时空图，算加速比。

**课程对应：** Ch5 习题 12（超标量/超流水线）。

**优点：** 和 Ch5 压轴题一致；有难度、有亮点。

**工作量：** 偏大；建议在完成基础五级流水后再做。

---

### ⭐ 第三梯队：偏理论仿真，硬件改动小

#### 7. 阵列机互连网络仿真（Cube / PM2I / Shuffle）
**做什么：** 用 Python/C++ 仿真 N=8 或 16 的互连网络；实现一步/多步路由；可选简单 SIMD 阵列做向量加。

**课程对应：** Ch6 习题 4、5（互连函数、节点连接）。

**优点：** 几乎不动现有 CPU；适合「向量/阵列机」但不想改硬件的人。

**缺点：** 偏软件仿真，和现有课设延续性弱；老师可能更认硬件/Verilog。

---

#### 8. 非线性流水线调度分析 + 小型多功能流水线
**做什么：** 设计 3–4 段多功能静态流水线（如 ADD/MUL/DIV 段数不同），求禁止表 F、冲突向量 C、最小平均间隔、画预约表。

**课程对应：** Ch5 习题 10、11（非线性流水线，必考）。

**优点：** 和考试高度一致；可只做分析 + 小规模 RTL。

**缺点：** 单独做偏「算题课设」，需配仿真或时空图才完整。

## 对你提到的三个方向的说明

| 方向 | 是什么 | 建议 |
|------|--------|------|
| **向量/阵列机** | 向量机：一条指令处理一整组数据（CRAY-1）；阵列机：大量 PE + 互连网络并行计算 | 做「**简易向量扩展**」（题目 3）或「**互连网络仿真**」（题目 7），不要一上来做完整阵列机 |
| **结合嵌入式** | CPU + 外设 + 裸机程序，形成可演示的小系统 | 选题目 5，或在题目 1/2 上加 UART/GPIO |
| **加 Cache** | 在 CPU 和主存间加速局部性访问 | 选题目 2，和现有主存接口最匹配 |

## 组合方案（若想既有深度又好写报告）

| 方案 | 组合 | 适合人群 |
|------|------|----------|
| **A（最稳）** | 五级流水 + 数据/控制冒险 | 想巩固 CPU、对齐 Ch5、面试友好 |

## 选题建议（结合你的情况）

1. **若课设周期约 3–4 周、还要复习考试** → 优先 **方案 A（五级流水线）** 或 **方案 B（加 Cache）**，和现有斐波那契程序衔接最好。
2. **若老师强调「体系结构」而非「组成原理」** → 在 A/B 基础上加 **性能报告**：时空图、吞吐率、加速比、Amdahl（访存/分支占比），直接对应 Ch1 + Ch5。
3. **若想和组里差异化** → **C（嵌入式 SoC）** 或 **D（简易向量）** 更容易讲出亮点。
4. **「向量处理机/阵列机」若只做理论** → 互连网络仿真可以当**报告一章**，不宜单独当整个课设，除非老师明确允许纯仿真。

## 报告里建议覆盖的体系结构考点（加分项）

无论选哪个题，答辩/报告里带上这些，会和课程高度一致：

- **透明性**：Cache、流水线寄存器对程序员是否可见  
- **软硬件分工**：冒险检测用硬件还是编译器调度  
- **并行性分类**：Flynn（SIMD 向量 / MIMD 多核）  
- **性能公式**：加速比 S、效率 E、吞吐率 TP  
- **与 Amdahl 的联系**：优化流水线后，访存/分支成为新瓶颈
