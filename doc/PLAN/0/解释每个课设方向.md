## 一、你现在这个 CPU 是什么（对照你的三张图）

### 图1：系统框图（单总线 + 微程序控制）

可以把它想成一条**单车道公路**：

```
PC / AC / AX / BX / CX / MDR …  →  【internal_bus 单总线】  →  MAR → RAM
                                      ↑
                                   ALU 也在旁边接这根线
                                      ↑
                              微程序控制器（大脑）
```

- **单总线**：一个时钟周期，总线上只能传**一份**数据（像单车道，一次只能过一辆车）。
- **微程序控制器**：每条机器指令（如 `ADD`）被拆成很多**微步骤**（取 MAR、读内存、ALU 运算……），一步步执行。
- **不是五级流水线**：你的波形里有 `microCommands`，说明是「一条指令跑完多个微周期」，**不是** IF/ID/EX/MEM/WB 五段同时跑 5 条指令。

### 图2：状态图（微程序在干什么）

上面一段是**公共取指**（00000→00001→00010→00011），所有指令都要走。

下面按 **OpCode** 分叉，例如：

- `MOVE AC,[BX]`：先 MAR←BX，再读内存，再 AC←MDR  
- `ADD AC,[BX]`：读内存 + ALU 加  
- `JNZ`：看 `PSW_Z_flag`，决定 PC 跳不跳  

**一条指令 = 多个状态 = 多个时钟周期**，顺序执行。

### 图3：波形（斐波那契在跑）

- `debug_BX_reg`：指向当前用到的内存地址（8、13…）  
- `debug_AC_reg`：累加结果（最后 13 = f₇）  
- `debug_CX_reg`：循环还剩几次  
- `microCommands`：每个周期控制器发出的微命令  

**结论**：这是一个能算斐波那契的**多周期、微程序、单总线 CPU**，属于**组成原理**典型课设。  
**计算机体系结构**课设，通常是在此基础上加：**并行、存储层次、I/O、性能分析** 等「更大一层」的东西。

## 二、逐个解释课设方向（都对照「你的 CPU」）

---

### 方向 1：五级流水线 CPU

**一句话**：让 5 条指令**同时在不同阶段**执行，像工厂流水线，而不是一条做完再做下一条。

**你现在**：

```
指令1: [取指][译码][执行][访存][写回]  全部做完
指令2:                              才开始...
```

**改成流水线后**：

```
周期1: 指令1取指
周期2: 指令1译码 | 指令2取指
周期3: 指令1执行 | 指令2译码 | 指令3取指
...
```

**要额外做啥**：

- 每级之间加**流水线寄存器**（保存上一级结果）  
- 处理**数据冒险**（上一条还没写回，下一条就要用）→ 转发 Forwarding  
- 处理**控制冒险**（`JNZ` 跳转）→ 冲刷流水线或停顿  

**和斐波那契的关系**：同一段斐波那契程序，对比「多周期版 vs 流水线版」谁周期数更少。

**课程对应**：复习计划第 5 章——时空图、吞吐率、加速比。

**难度**：中等，最推荐，和现有 CPU 衔接最好。

---

### 方向 2：加 Cache（高速缓存）

**一句话**：在 CPU 和 RAM 之间加一块**小但快**的内存，常用数据放这里，少等 RAM。

**你现在**（图1）：

```
CPU → 直接访问 RAM（每次访存都慢）
```

**加 Cache 后**：

```
CPU → 先查 Cache
        ├─ 命中：很快拿到数据
        └─ 缺失：再去 RAM 读一整块（cache line），顺便放进 Cache
```

**要额外做啥**：

- 地址分成：tag | index | offset  
- 命中/缺失判断  
- 替换策略（如 LRU）、写策略（写回/写直达）  
- 统计命中率  

**和斐波那契的关系**：斐波那契反复读 `Mem[BX]`、`Mem[BX+1]`，局部性较好，加 Cache 后访存变快；可以对比加/不加 Cache 的总周期。

**课程对应**：第 3 章存储、第 1 章「透明性」（程序一般感觉不到 Cache 存在）。

**难度**：中等；**不用大改**你的单总线和微程序，主要在 CPU 和 RAM 之间插一层。

---

### 方向 3：向量处理机 / 阵列机（你不太清楚的那个）

先分开说，这是两个东西：

#### 3a. 向量处理机（SIMD 思路）

**一句话**：一条指令，同时算**一整组数**，不是一次只算一个。

**你现在**：

```
算 a[0]+b[0]  → 一条 ADD
算 a[1]+b[1]  → 再一条 ADD
算 a[2]+b[2]  → 再一条 ADD
...
```

**向量版**：

```
VADD VR0, VR1, VR2   // VR0[i] = VR1[i] + VR2[i]，i=0..7 一次做完
```

**要额外做啥**：

- 向量寄存器 VR0、VR1…（每个存 4 或 8 个数）  
- 向量加/乘/load/store 几条指令  
- （可选）链接：上一条向量结果直接给下一条用（类似 CRAY-1，课程第 6 章）

**和斐波那契的关系**：可以一次生成多个斐波那契项，或做数组相加 demo。

**难度**：中偏大；比完整阵列机简单。

#### 3b. 阵列机（Array Processor）

**一句话**：很多个**相同的小计算单元 PE** 排成阵列，各自算一部分，再通过**互连网络**交换数据。

```
PE0  PE1  PE2  PE3
PE4  PE5  PE6  PE7   ← 很多核同时算
  \   |   |   /
   互连网络（Cube、Shuffle 等，第 6 章作业）
```

**特点**：硬件规模大，课设里常做成**仿真**（Python 模拟 8 个 PE 做向量加），而不是在现有 CPU 上硬改。

**建议**：若选这个方向，做「**简易向量扩展**（3a）」更现实；阵列互连可写报告一章，不必整块硬件都做。

---

### 方向 4：结合嵌入式（SoC / 外设）

**一句话**：你的 CPU 不只会算斐波那契，还能**控制外部设备**（灯、串口、按键）。

**你现在**：

```
CPU ←→ 只有 RAM
```

**嵌入式版**：

```
CPU ←→ RAM
  ↕
GPIO（LED 亮灭）
UART（串口打印结果）
Timer（定时中断）
按键（读 n 是几）
```

**要额外做啥**：

- **内存映射 I/O**：例如地址 `0x1000` 不是 RAM，是「LED 寄存器」  
- 写裸机程序：按键输入 n，算完斐波那契，串口/数码管输出  
- （可选）定时器中断  

**和斐波那契的关系**：算完把 13 送到串口显示，或按按键改 n，**答辩演示很直观**。

**课程对应**：第 3 章 I/O、通道；软硬件分工。

**难度**：中等；数据通路改动不大，主要是加外设模块和地址译码。

---

### 方向 5：中断系统

**一句话**：CPU 正在算斐波那契时，**外部事件**（定时器、按键）可以**打断**它，处理完再回来。

**你现在**：程序从头跑到尾，不会被打断。

**加中断后**：

```
正在算斐波那契…
  ↓ 定时器响了！
保存 PC、寄存器 → 去中断服务程序 → 处理完 → 恢复 → 继续算
```

**要额外做啥**：

- 中断请求线、中断屏蔽位（课程第 3 章大题常考）  
- 优先级  
- 现场保存/恢复  

**和嵌入式的关系**：嵌入式几乎都会用到中断；可以和中断 + 串口一起做。

**难度**：中等；控制逻辑变复杂，但单总线结构可以保留。

---

### 方向 6：超标量 / 双发射（进阶流水线）

**一句话**：流水线已经会了，再进一步——**一个周期发射 2 条**无相关指令。

**难度**：偏大；建议先做完方向 1 再考虑。

---

### 方向 7：互连网络仿真（偏软件）

**一句话**：不写 Verilog，用程序模拟 Cube、PM2I、Shuffle 等网络，算「节点 i 一步连到谁」。

**特点**：和第 6 章作业题一致，但和**你的 CPU 图几乎无关**，延续性弱。

**适合**：时间紧、不想改硬件，老师允许纯仿真报告。

## 三、你提的三个方向，用最直白的话总结

| 你说的是 | 实际意思 | 要改你 CPU 的哪里 |
|---------|---------|------------------|
| **加 Cache** | CPU 和 RAM 之间加快取 | 在 MAR/RAM 之间插 Cache 模块，改访存路径 |
| **向量/阵列机** | 一次算很多数 / 很多核并行 | 加向量寄存器+宽 ALU；阵列机通常另做仿真 |
| **结合嵌入式** | CPU 能点灯、串口输出 | 加 GPIO/UART，用地址区分 RAM 和外设 |
