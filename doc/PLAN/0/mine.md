## 四个扩展方向，逐个说「值不值得做」

### 1. Cache —— **必做（第二优先级）**

**就业价值：高**

- 后端性能优化、JVM/Go 运行时、数据库 buffer pool，本质都是「缓存层次」
- 面试常问：为什么顺序访问快、Cache line、false sharing（你哪怕仿真里只测 hit rate 也够）

**和你 CPU 的关系：**

```
现有：CPU → MAR → RAM
升级：CPU → MAR → Cache Controller → RAM
```

**建议规模（5–7 天）：**

- 只做 **D-Cache**，直接映射或 2 路组相联  
- 16/32 行 × 4 word 即可  
- 统计：hit rate、miss penalty、斐波那契总周期对比  

**简历一句话：**  
「在 VHDL 五级流水 CPU 上实现直接映射 D-Cache，对比有无 Cache 下斐波那契程序 CPI。」

---

### 2. 中断 —— **强烈建议做（第三优先级）**

**就业价值：高（尤其你有 OS 内核）**

- 把「硬件课设」和「OS 课设」连成完整链路，简历非常好讲  
- B 端设备、驱动、网络栈、定时器，都绕不开中断  

**建议规模（4–5 天）：**

- 1 个 **Timer 中断** + 1 个 **外部按键/串口中断** 即可  
- 实现：中断向量表、屏蔽位、现场保存 PC/部分寄存器  
- 演示：斐波那契跑着跑着，定时器打断，计数 +1 或切换任务（哪怕只是打印标志位）

**简历一句话：**  
「实现 RISC 风格异常/中断入口，与 OS 课设中的 trap handler 概念对应。」

---

### 3. 嵌入式（外设）—— **做「最小可演示」即可（第四优先级）**

**就业价值：中–高（看岗位）**

- 投 **嵌入式 / FPGA / 工控 B 端**：价值高  
- 投 **纯 Java/Go 后端**：价值中等，但演示效果好  

**不要做大而全 SoC**，只做：

- **UART** 输出斐波那契结果（必做）  
- **GPIO/LED** 或按键改 n（二选一）  
- 内存映射 I/O：`0xFFFF0000` = UART 数据寄存器  

**建议规模（3–4 天）：**  
可以和中断合并——按键触发中断，串口输出结果。

**简历一句话：**  
「内存映射 I/O，UART 裸机输出，支持按键中断修改计算参数。」

---

### 4. 向量 —— **报告级，不必完整 RTL（最低硬件优先级）**

**就业价值：对 B 端后端偏低**

- 除非投芯片/AI 加速/NPU 方向  
- 完整向量寄存器 + 链式执行，工作量和流水线 + Cache 差不多，但面试命中率低  

**建议做法（1–2 天，写报告）：**

- 用 Python/Excel 做 **4 元素向量加** 的时空图  
- 算标量 vs 向量加速比  
- 对照课程第 6 章 CRAY-1 链接、互连网络  
- 答辩时说：「硬件资源有限，向量部分做性能建模；若扩展可在 EX 段加 SIMD 数据通路」

## 推荐实施顺序（总约 3–4 周）

```
第1周    五级流水线（核心）
         ├─ IF/ID/EX/MEM/WB + 流水线寄存器
         ├─ 数据转发 + load-use stall
         └─ 分支 flush（先不做分支预测）

第2周    D-Cache
         ├─ 直接映射，16 行
         └─ 性能对比报告（有/无 Cache）

第3周    中断 + 最小嵌入式
         ├─ Timer IRQ + UART 输出
         └─ 可选：按键改 n

第4周    向量（报告）+ 整合答辩
         ├─ 1 页向量加速比分析
         └─ 整合 testbench、波形、文档
```

**如果只有 2 周：** 流水 + Cache，中断和 UART 二选一。  
**如果有 4 周+：** 按上表全做，向量只写报告。

---

## 为什么不是「向量优先」或「嵌入式做大」

| 方向 | 你现在的顾虑 | 建议 |
|------|-------------|------|
| 向量先做 | 和现有单总线 CPU 改动大，和 B 端后端关联弱 | 放最后，报告即可 |
| 嵌入式做大 | LED/数码管/多外设耗时间，答辩边际收益递减 | UART + 1 个中断够用 |
| Cache 跳过 | 以为「只是组成原理」 | 体系结构 + 后端性能都考这个 |
| 中断跳过 | 以为和 CPU 无关 | 你有 OS 内核，这是最大差异化 |

**若投嵌入式/FPGA B 端：** 强调 UART、中断、FPGA 综合时序。
