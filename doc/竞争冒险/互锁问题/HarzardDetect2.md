 **load-use 检测 + 插 bubble（stall）**

当 **EX 级是一条 Load（`Mem_read=1`）**，且 **ID 级这条指令马上要读** 的寄存器，正好是那条 Load 的 **目的寄存器 `rd`** 时，硬件 **不能只靠转发**，必须 **停一拍 + 在 EX 插 NOP（bubble）**。

Hazard Unit 就是：**发现这种情况 → 发出 stall / bubble 控制信号**。

---

## 输入（4 个）——只用来“比寄存器号 + 看是不是 Load”

| 信号           | 来自哪里         | 含义                                                      |
| ------------ | ------------ | ------------------------------------------------------- |
| **IF/ID.RS** | IF/ID 流水线寄存器 | **当前在 ID 级** 的指令要用的 **源寄存器 1 编号**                       |
| **IF/ID.RD** | IF/ID        | **当前在 ID 级** 的指令要用的 **源寄存器 2 编号**                       |
| **ID/EX.RD** | ID/EX        | **当前在 EX 级**（上一条已进入 EX）的指令的 **目的寄存器 rd**（Load 要写进哪个寄存器） |
| **Mem_read** | ID/EX 里的控制位  | **当前在 EX 级** 的指令是不是 **读内存**（`LW`/`LD` 时为 1）             |

**检测逻辑：**

```text
load_use =
    Mem_read
    AND (ID/EX.RD ≠ 0)
    AND (ID/EX.RD = IF/ID.RS  OR  ID/EX.RD = IF/ID.RD)
```

含义用一句话说：

> **前面是 Load，后面这条在 ID 里就要读 Load 的目标寄存器 → 必须 stall。**

这些输入 **不送数据**，只送 **寄存器编号 + 控制位** 做比较。

---

## 输出

### 1. `Ifid_en`（IF/ID 使能）

| 值     | 效果                                                         |
| ----- | ---------------------------------------------------------- |
| **1** | 正常：每个时钟 IF/ID 更新，ID 里的指令进入下一级                              |
| **0** | **冻结 IF/ID**：ID 里那条指令 **多停 1 拍**，IF 也不再往下推进（常配合 `Pc_en=0`） |

对应 load-use：**让“要用 Load 结果的那条”停在 ID**，别进 EX。

---

### 2. `Control_src`（控制信号来源选择）

接到 **Control 与 ID/EX 之间的 MUX**：

| 值 | MUX 选谁 | 效果 |
|----|----------|------|
| **1**（正常） | 来自 **Control** 的真实控制信号 | 正常指令进入 ID/EX |
| **0**（ hazard） | 常数 **全 0** | 向 **ID/EX** 插入 **bubble（NOP）**：WB/MEM/EX 控制全 0 → EX 级空转一拍 |

也就是：**EX 段插入气泡**，给前面的 Load 多一拍，等 MEM 读完再算。

---

### 3. `Pc_en`

与 `Ifid_en` 配合：

| 值 | 效果 |
|----|------|
| **0** | **PC 不更新** → IF 不再取新指令，避免流水里塞进多余指令 |

三者一起才是完整 stall：

```text
Pc_en = 0        → 不取新指令
Ifid_en = 0      → ID 里那条停住
Control_src = 0  → ID/EX 进 NOP（bubble）
```

---

## 对应 LW 例子（周期 4 为何要 bubble）

```text
周期 4:  LW 在 MEM（Mem_read=1，rd=R2）
         AND 在 ID（IF/ID.RS 或 RD = R2）

Hazard 成立 → Ifid_en=0, Control_src=0, Pc_en=0
→ AND 留在 ID，EX 里是 NOP
→ 周期 5: LW 数据就绪，AND 才进 EX，此时可再靠 Forward
```

---

## 一句话记

| 类型 | 含义 |
|------|------|
| **输入** | “ID 这条要读谁？”和“EX 那条 Load 要写谁？”是否冲突，且 EX 是否为 Load |
| **输出** | **别往前取指、别让 ID 往下走、往 EX 塞一条全 0 控制的 NOP** |

若你还画了 **分支 flush** 的线（`branch_taken` 等），那是 Hazard Unit 的另一块逻辑；你这张图主要是 **load-use stall** 这一路。需要的话可以把分支那几路输入输出也单独列一张表。
