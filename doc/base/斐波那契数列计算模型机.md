# 🌱原理卡：微程序控制器版

> [!NOTE] 
> CPU功能：计算斐波那契数列的第n项fn (0-based：n=0 -> f1, n=1-> f2)

---

## 关键点
### 设计思路

1. **架构基础：** 采用经典的**冯·诺依曼结构**，其中主存储器 (`main_memory`) 存储指令和数据。
2. **数据通路 (`datapath`)：**
    - 核心是一个**单总线**结构 (`internal_bus`)，所有寄存器（PC, MAR, MDR, AC, AX, BX, CX）通过多路选择器连接到该总线。
    - 指令通过一系列**微命令**（如 `LoadPC`, `LoadMAR`, `LoadAC`, `IncBX`, `MemW` 等）精确控制数据在总线上的流动和寄存器的更新。
    - 包含一个简单的 **ALU** 用于执行算术逻辑操作（加/减/与）。
    - 提供状态反馈（OpCode 和 `PSW_Z_flag`）给控制器。
3. **控制单元 (`microController`)：**
    - 采用**微程序控制**方式，控制逻辑存储在**控制存储器 (CM_ROM)** 中。
    - 执行流程为：**取指 (Fetch)** -> **译码/映射 (Map)** -> **执行 (Execute)**。
    - **取指阶段**：固定地址序列（0-3）。
    - **译码阶段**：通过指令的 OpCode 进行**映射 (Map)**，跳转到 CM_ROM 中对应指令执行微程序的起始地址。
    - **执行阶段**：微指令序列依次执行。支持**条件分支** (`JNZ` 依赖 `PSW_Z_flag`)，用于实现循环控制（如斐波那契数列的循环计算）。
4. **功能实现：** 通过预先加载到主存中的机器指令，实现了斐波那契数列的计算（LOAD、MOVE、ADD、INC、DEC、JNZ 等指令）。

LOOP:

2：MOVE AC, [BX] ;AC <- M[BX] (读出ALU_A，装到AC)

3：INC BX ; BX = BX + 1（指向 ALU_B）

4：MOVE AX, BX ; AX = BX

5：ADD   AC,[BX] ;AC = AC + M[BX],（ALU_A + ALU_B）AC + M[BX] -> M[AX]

6：INC  AX（指向 ALU_out）

7：STORE  AC, M[AX] ;把AC中的结果存到AX指定的地址

8：DEC  CX ;CX = CX - 1

9：JNZ  LOOP ;if CX != 0, 跳回LOOP（继续计算）

### 指令流（多周期架构）

本图展示了 CPU 周期（T 状态）与微操作（Micro-ops）的对应关系，遵循同步时序逻辑：**控制信号在 $T_n$ 期间准备好，并在 $T_{n+1}$ 的时钟沿触发动作。**

---

#### I. 取指阶段（Fetch Cycle）- 4 T 周期

|周期 (T State)|源寄存器 (Source)|目标寄存器 (Destination)|控制信号 (Control)|目的|
|:--|:--|:--|:--|:--|
|**$T_1$**|PC|MAR|`LoadMAR='1'`, `Bus_Sel="PC"`|将指令地址送入 MAR。|
|**$T_2$**|RAM (Mem[MAR])|MDR|`MemRead='1'`, `LoadMDR='1'`|触发内存读操作，MDR 锁存 Opcode。|
|**$T_3$**|MDR|IR|`LoadIR='1'`, `PC $\leftarrow$ PC+1`|IR 锁存 Opcode。PC 指向下一个地址/操作数。|
|**$T_4$**|PC|MAR|`LoadMAR='1'`, `Bus_Sel="PC"`|将操作数地址送入 MAR。|
|**$T_5$**|RAM (Mem[MAR])|OPR|`MemRead='1'`, `LoadMDR='1'`, `MDR $\rightarrow$ OPR`|OPR 锁存 Operand/Address。|

_(注：在 $T_5$ 结束后，指令 $\text{IR}$ 和操作数 $\text{OPR}$ 已经准备好。)_

---

#### II. 译码阶段（Decode Cycle）

|阶段 (State)|动作 (Action)|
|:--|:--|
|**$T_6$**|**组合逻辑：** 控制器读取 $\text{IR}$ (Opcode) 和 $\text{OPR}$ (Operand/Address)。通过 MAP 逻辑，确定下一微指令地址，跳转到对应的执行子程序。|

---

#### III. 执行阶段（Execute Cycle）- 变量 T 周期

##### A. 内存访问 (Load/Store)

|周期 (T State)|示例操作 (e.g., $\text{LOAD AC, [BX]}$)|
|:--|:--|
|**$T_{Exec_1}$**|地址计算/准备：`MAR $\leftarrow$ Addr / BX`。|
|**$T_{Exec_2}$**|内存操作：`MemRead/MemWrite='1'`。 (触发内存动作)|
|**$T_{Exec_3}$**|数据传输：`MDR $\leftarrow$ MemData / MemData $\leftarrow$ MDR`。 (MDR 锁存)|
|**$T_{Exec_4}$**|最终写入：`AC $\leftarrow$ MDR / R $\leftarrow$ MDR`。|

##### B. ALU 操作 (ADD $\text{AC}, \text{MDR}$)

|周期 (T State)|示例操作 (e.g., $\text{AC} \leftarrow \text{AC} + \text{MDR}$)|
|:--|:--|
|**$T_{Exec_1}$**|ALU 输入准备：`ALU A $\leftarrow$ AC`；`ALU B $\leftarrow$ MDR`。|
|**$T_{Exec_2}$**|结果写入：`AC $\leftarrow$ ALU\_out`；`LoadPSW`。|

#### 💡 核心时序原理（同步时序）

| 概念                    | 动作发生时间                              | 关键点                                                                                                                                        |
| :-------------------- | :---------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------- |
| **微命令 $\mu$ Command** | $T_n$ **到** $T_{n+1}$ **之间** (组合逻辑) | 控制信号（如 $\text{LoadMAR}$）由当前 $\mu\text{IR}$ 产生，必须在 $T_{n+1}$ 时钟沿前稳定。                                                                        |
| **寄存器更新**             | $T_{n+1}$ **时钟上升沿** (时序逻辑)          | 寄存器（如 $\text{MAR, PC, AC}$）的值在 $\mu\text{Command}$ 信号的触发下，在 $T_{n+1}$ **瞬间**捕获数据。                                                          |
| **微程序控制**             | $T_{n+1}$ **时钟上升沿** (时序逻辑)          | 微地址寄存器 ($\mu\text{AR}$) 和微指令寄存器 ($\mu\text{IR}$) 更新状态，控制流进入 $T_{n+1}$ 状态。                                                                  |
| **数据同步延迟**            | **必须等待一个周期**                        | $\text{MAR} \leftarrow \text{PC}$ 这个操作在 $T_n$ 期间触发，但在 $T_{n+1}$ **之后** $\text{MAR}$ 的新值才稳定可用。任何需要使用 $\text{MAR}$ 新值的操作必须安排在 $T_{n+2}$ 或之后。 |
| **IR 的作用**            | 永久锁存                                | $\text{IR}$ 锁存当前指令，确保在 CPU 忙于访存数据时（如 $T_4$ 以后），控制器仍能通过 $\text{IR}$ 获取正确的 Opcode。                                                           |

## 💡总结

### 一、 设计中体现的核心知识点

#### 1. 计算机组成原理核心概念

|模块|关键知识点|说明|
|:--|:--|:--|
|**`datapath` (数据通路)**|**单总线结构 (Single Bus)**|所有数据传输都通过一根共享总线 (`internal_bus`)，简化布线，但牺牲了速度（一个时钟周期只能完成一次数据传输）。|
||**寄存器组织 (Registers)**|实现了通用寄存器 (AX, BX, CX)、专用寄存器 (PC, MAR, MDR, IR, AC) 的功能，并理解它们各自的用途。|
||**ALU 功能 (Arithmetic Logic Unit)**|实现了基本的算术和逻辑操作，并且理解如何通过多路选择器（MUX）选择 ALU 的输入 (`ALU_A`, `ALU_B`)。|
||**时序逻辑 (Sequential Logic)**|寄存器的值在**时钟上升沿**、并且在**微命令（Load信号）有效**时才更新。这是同步电路的基础。|
||**组合逻辑 (Combinational Logic)**|总线选择器 (`Bus_Sel`)、PC 的下一个值选择器 (`PC_Sel`)、ALU 运算等都是组合逻辑，它们的输出**立即**随输入变化。|
|**`main_memory` (主存储器)**|**RAM 模型 (RAM Model)**|实现了简单的同步写入 (Write-Enabled, Clocked) 和异步读取 (Address-driven)。|
|**`microController` (控制单元)**|**微程序控制 (Microprogrammed Control)**|控制信号不是硬编码的逻辑门，而是存储在 **ROM (CM_ROM)** 中的数据字（即**微指令**）。|
||**取指-译码-执行周期 (Fetch-Decode-Execute)**|实现了 CPU 运行的基本三步骤：**0-3 步取指**，**第 4 步映射/译码**，**5 步及以后执行**。|

#### 2. 微指令格式及控制字段

 32 位微指令

|字段（位）|名称|功能/作用|
|:--|:--|:--|
|`31-19`|**控制信号 (MicroCommands)**|直接控制数据通路中的 Load 信号、MemW 信号等。例如：`LoadPC`、`LoadMAR`、`MemW`。|
|`18-16`|**总线选择 (`Bus_Sel`)**|控制哪个寄存器的内容驱动到 `internal_bus`。**这是单总线 CPU 调度的核心**。|
|`15-14`|**PC 选择 (`PC_Sel`)**|控制 PC 的下一值来源：保持、PC+1、或从总线加载（用于跳转）。|
|`13-10`|**数据目标选择**|如 `MDR_Sel` 和 `AC_Sel`，控制 MDR 或 AC 的数据来源（来自总线、RAM 或 ALU）。|
|`9-7`|**ALU 操作 (`ALU_op`)**|控制 ALU 执行何种操作（ADD, SUB, AND）。|
|`6-5`|**分支条件 (`branch_conditions`)**|决定下一条微指令的地址来源（顺序、跳转、条件跳转、映射）。|
|`4-0`|**下地址字段 (`NextAddress`)**|指定下一个微地址，用于跳转或条件分支。|

### 二、 **关键设计思想和权衡**

这个模型是一个简化版本，它体现了在设计一个实际 CPU 时需要做的**权衡 (Trade-offs)**。

|设计思想|描述|权衡/取舍|
|:--|:--|:--|
|**微程序控制 vs. 硬布线控制**|您的设计采用**微程序控制**，将控制逻辑固化到 ROM 中。|**优点：** 设计灵活，易于修改和扩展指令集。**缺点：** 速度较慢，因为每次执行微指令都需要读取 ROM。现代高性能 CPU 多用**硬布线控制**（速度更快）。|
|**单总线 vs. 多总线**|您的设计是**单总线**。|**优点：** 硬件成本低，布线简单。**缺点：** 性能受限，许多操作需要多个时钟周期完成（例如，`MAR <= PC` 和 `PC <= PC + 1` 不能在一个周期内同时完成）。|
|**指令周期分解**|无论是任何指令，都遵循**取指 (Fetch)** 和**执行 (Execute)** 两大阶段。|**关键学习点：** 必须能将高级指令（如 `ADD AC, [BX]`）分解成一系列不可再分的**微操作**（如 `LoadMAR`, `Bus_Sel` 等），并安排它们在不同时钟周期执行。|
|**时序控制**|严格区分**同步 (Sequential)** 和**组合 (Combinational)** 逻辑。|**关键学习点：** 只有寄存器是同步元件，其变化受 `clk` 控制；而总线、ALU、选择器都是组合元件，其变化是即时的。这是保证数据同步传输和处理的基础。|
|**条件控制**|理解如何通过状态标志位 (`PSW_Z_flag`) 和条件分支逻辑 (`branch_conditions`) 实现程序流程控制（如 `JNZ` 循环）。|**关键学习点：** 控制器根据数据通路的**状态反馈**（Z 标志）来决定下一步的控制流，这是实现复杂指令的基础。|

### 汇报
老师好，我们组的 CPU 功能是计算斐波那契数列的第 n 项 ($f_n$)。  
首先在指令系统中，我们设计了多条基本指令，包括 LOAD、MOVE、ADD、INC、DEC、STORE 和 JNZ，用来实现循环计算。程序会把 ($f_1$) 和 ($f_2$) 存放在内存 11 和 12 号地址，接着通过循环求出后面的项，并依次写回内存。

在结构上，我们的 CPU 采用**冯·诺依曼结构**，主存储器（`main_memory`）同时存放指令和数据。  

数据通路（`datapath`）为**单总线结构**，所有寄存器通过总线进行数据传输。其中，`BX` 指向当前项，`CX` 存放循环次数（即 _n - 2_）。

控制单元（`microController`）采用**微程序控制方式**，通过控制存储器（`CM_ROM`）中的微指令依次完成取指、译码和执行全过程。  
每条机器指令被分解为若干微操作，由微指令精确控制寄存器的读写和 ALU 的运算过程。

在控制流程上，`μIR` 用于存放当前微指令，`μAR` 指向下一条微指令地址。  
取指阶段由一段固定的微指令序列完成；译码阶段通过映射（`MAP`）将操作码跳转到对应微程序入口；执行阶段则按微指令顺序运行。  
其中，`JNZ` 指令的循环控制由微指令的条件分支字段与零标志位（`PSW_Z`）共同决定，当结果不为零时跳回循环入口，从而实现整个斐波那契数列的迭代计算。

我们在仿真时设置计算第七项，也就是 ($f_7$)。在波形图中可以看到，随着循环进行，`CX` 从 5 逐步减到 0，每次 `STORE` 信号有效时都会把新的斐波那契项写入内存。  
最后一次循环结束后，`AC_reg` 的值为 **13**，对应的就是第七项的结果，说明整个指令流程和循环逻辑是正确的。

### 微程序控制器
**在一个时钟周期内发生**：

1. **取指地址 (uAR):** 当前微地址寄存器 (`uAR_reg`) 指向控制存储器（CM/ROM）。
2. **取微指令 (CM/cm_output):** 从 CM 中读取当前微指令。
3. **执行微指令 (uIR/microCommands):** 将读取的微指令发送给数据通路，控制硬件执行操作。
4. **确定下一地址 (next_uAR):** 根据当前指令的控制字段和条件（如 `PSW_Z_flag`），计算出下一条微指令的地址。
