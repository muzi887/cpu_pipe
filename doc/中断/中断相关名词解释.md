# 中断相关名词解释

> 本课设中断实现见 [Timer精确中断实现.md](./Timer精确中断实现.md)  
> 对应 RTL：`interrupt_controller.vhd`、`timer.vhd`、`cpu_top.vhd`、`soc_top.vhd`

---

## 1. 系统与架构

### SoC — System on Chip（片上系统）

把 **CPU、Cache、主存、外设** 集成在一个顶层模块里的完整系统。

本项目中 `soc_top.vhd` 即最小 SoC：

```text
CPU + I-Cache + D-Cache + main_memory + Timer + UART + GPIO
```

### MMIO — Memory-Mapped I/O（内存映射 I/O）

把外设寄存器映射到**内存地址空间**，CPU 用普通的 `LD` / `ST` 访问外设，而不是专用 I/O 指令。

| 地址 | 外设 | RTL |
|------|------|-----|
| `0xFF00` | UART 数据 | `uart_mmio.vhd` |
| `0xFF04` | UART 状态 | `uart_mmio.vhd` |
| `0xFF10` | GPIO LED | `gpio_mmio.vhd` |
| `0xFF20` | Timer 控制（`ADDR_CTRL`） | `timer.vhd` — **`0xFF20(0)`=使能，`0xFF20(1)`=重载** |
| `0xFF24` | Timer 周期（`ADDR_PERIOD`） | `timer.vhd` |

`soc_top` 中 `mem_addr(15:8) = 0xFF` 时判定为 MMIO，**旁路 D-Cache**，直接访问外设。

#### MMIO 总线信号（外设端口常见命名）

CPU 执行 `ST` / `LD` 时，经 `soc_top` 连到外设的信号：

| 信号 | 方向 | 对应指令 | 含义 |
|------|------|----------|------|
| **`addr`** | CPU → 外设 | 两者 | 访问哪个 MMIO 寄存器（如 `0xFF20`） |
| **`wdata`** | CPU → 外设 | `ST` | **写数据**：Store 要写入的 16 位数（CPU 侧常叫 `mem_wdata`） |
| **`write_en`** | CPU → 外设 | `ST` | 本拍是否执行写 |
| **`rdata`** | 外设 → CPU | `LD` | **读数据**：Load 读回的 16 位数（Timer 见 §7 **rdata 读回**） |
| **`read_en`** | CPU → 外设 | `LD` | 本拍是否执行读 |
| **`sel`** | 译码 | — | 本拍访问是否选中该外设（地址匹配） |

**`wdata` 不是每拍都参与倒计时**；只有 CPU 写 MMIO 寄存器时，外设才用 `wdata` 更新内部寄存器（见 §7 Timer）。

### 外设（Peripheral）

CPU 之外的功能模块，如 Timer、UART、GPIO。本课设通过 MMIO 与 CPU 相连。

#### UART — Universal Asynchronous Receiver/Transmitter（通用异步收发器 / 串口）

**作用**：CPU 与外部设备按**串行**方式交换数据，常见于打印调试信息、与 PC/传感器通信。本课设实现为**简化 MMIO 模型**，不做真实波特率收发，仅把 CPU 写入的数据**锁存**起来供仿真观察。

| 地址 | 寄存器 | 访问 | 行为 |
|------|--------|------|------|
| `0xFF00` | UART_DATA | 写 | `ST` 写入的数据锁存到 `uart_data_reg` |
| `0xFF04` | UART_STATUS | 读 | 恒返回 `0x0001`（bit0=TX ready，表示可写） |

RTL：`uart_mmio.vhd`；SoC 调试口 `debug_uart_data` 可观察最近写入的数据。

典型用法：ISR 或主程序执行 `ST x3, 0(x7)`（`x7` 预载为 `0xFF00`），将斐波那契结果等输出到 UART。

#### GPIO — General Purpose Input/Output（通用输入输出）

**作用**：用少量引脚做**通用数字控制**，如点亮 LED、读取按键。本课设将 GPIO 实现为**写 LED 寄存器**：CPU 写入的 16 位数据直接驱动 `led_out`，仿真中可视为 LED 状态。

| 地址 | 寄存器 | 访问 | 行为 |
|------|--------|------|------|
| `0xFF10` | GPIO_LED | 写 / 读 | 写：更新 `led_reg`；读：读回当前 LED 值 |

RTL：`gpio_mmio.vhd`；SoC 调试口 `debug_led` 可观察 LED 寄存器。

与 UART 的区别：UART 侧重**串行数据输出**；GPIO 侧重**按位控制硬件引脚**（本项目中简化为一个 16 位 LED 寄存器）。

## 2. 中断基本概念

### 中断（Interrupt）

CPU 执行主程序时，**外设或 Timer 等异步事件**请求 CPU 暂停当前流程、转去处理紧急任务，处理完再返回。

本课设中断源：**Timer 定时溢出**（`timer.vhd` 产生 `irq_timer`）。

### 中断源（Interrupt Source）

产生中断请求的设备或事件。本项目唯一中断源为 **Timer**。

### IRQ — Interrupt Request（中断请求）

外设或 Timer 向 CPU **发出的“有事件要处理”请求**。本课设里 IRQ **不是 16 位数据**，而是 **单 bit 控制信号线**（如 `irq_timer`）；CPU 不通过 Load 读 IRQ，而是由硬件置 **pending** 并在条件满足时响应。

| 层次 | 本课设信号 | 含义 |
|------|------------|------|
| 外设发出 | **`irq_timer`** | Timer 溢出单拍脉冲（`timer.vhd`：`irq_timer <= irq_pulse`） |
| 控制器汇总 | **`irq_pending`** | 有未处理的中断请求（`interrupt_controller` 内部 `pending`） |
| CPU 响应 | **`irq_take`** | 本拍真正进入 ISR（还需 `IE=1`、`wb_commit` 等） |
| 确认清除 | **`irq_ack`** | 响应后清 pending（本项目 **`irq_ack <= irq_take`**） |

```text
Timer 溢出 → irq_timer（IRQ 线，1 拍）→ pending←1 → … → irq_take → ISR
```

命名习惯：RTL 里常以 **`irq_` 前缀** 表示与中断请求/响应相关的信号；**IRQ** 泛指这类请求本身，**`irq_pending`** 表示请求已被控制器**挂起**、等待 CPU 处理。

### 中断控制器（Interrupt Controller）

管理 CSR、汇总各中断源、维护 `irq_pending` 的模块。

对应 RTL：`interrupt_controller.vhd`。

### 单级中断

同一时刻只处理**一种、一级**中断，不支持高优先级中断打断正在执行的 ISR。本课设为单级模型。

### 中断响应（Interrupt Response / Take）

CPU 确认中断请求，保存现场、关中断、跳转 ISR 的过程。代码中对应信号 **`irq_take`**。

### 中断返回（Interrupt Return）

ISR 执行完毕后回到主程序。本课设通过 **`IRET`** 指令实现，对应信号 **`iret_commit`**。

## 3. CSR 与专用寄存器

### CSR — Control and Status Register（控制与状态寄存器）

CPU 内部用于**系统控制**的特殊寄存器，不属于通用寄存器堆 `x0`～`x15`。

由 `interrupt_controller.vhd` 实现，CPU 通过 SYS 类指令或中断硬件逻辑访问。

| CSR | 信号/寄存器 | 作用 |
|-----|-------------|------|
| **EPC** | `epc_reg` | 保存中断返回地址 |
| **STATUS.IE** | `status_reg(0)` | 全局中断使能（1=允许响应） |
| **CAUSE** | `cause_reg` | 记录中断原因（Timer 时 bit0=1） |

### EPC — Exception Program Counter（异常/中断程序计数器）

保存**被中断后主程序应继续执行的 PC**。

- 中断进入：`EPC ← if_pc`（`cpu_top.vhd` 中 `epc_wdata <= if_pc`）
- 中断返回：`PC ← EPC`（`IRET` 时 `pc_redirect <= epc_reg`）

### STATUS.IE / IE（Interrupt Enable，中断使能）

全局中断开关。`IE=0` 时即使有 `irq_pending` 也不响应；`EI` 置 1，`DI` 置 0；进入 ISR 时硬件自动关中断。

### CAUSE（中断原因寄存器）

记录**哪类事件**触发中断。本项目 Timer 中断写入 `CAUSE_TIMER = 0x0001`（bit0 表示 Timer）。

## 4. 程序与指令

### ISR — Interrupt Service Routine（中断服务程序）

中断发生后 CPU **跳转执行的程序段**。

本课设 ISR 入口 **`ISR_ADDR = 0x0100`**，预载于 `main_memory.vhd`：

```asm
ADDI x6, x6, 1    ; 0D81 — 中断计数 +1
IRET              ; 返回主程序
```

#### x6 — 中断计数寄存器（irq_cnt）

ISR 里 **`ADDI x6, x6, 1`** 每响应一次 Timer 中断就把 **x6 加 1**，用作**中断次数计数器**。

| 项目 | 说明 |
|------|------|
| 别名 | `irq_cnt`（中期报告寄存器约定） |
| 为何用 x6 | 斐波那契主程序占用 x1～x5；x6 空闲，ISR 修改不影响主程序 |
| 验收 | 仿真中 x6 随 Timer 周期性中断递增，证明 ISR 执行且 IRET 能返回 |
| 扩展 | 可将 x6 `ST` 到 UART（`0xFF00`）输出中断次数 |

机器码 `0x0D81` 预载于 `main_memory.vhd` 地址 `256`（`0x0100`）。

### ISR 入口 / 中断向量地址

中断发生时 PC 跳转的目标地址。本项目固定为 **`0x0100`**（`cpu_top` 常量 `ISR_ADDR`）。

### 主程序（Main Program）

正常运行、可被中断打断的程序。本项目为**斐波那契**（地址 `0x0000` 起），末尾 `EI` 开中断后 `HALT`。

### SYS 指令

OpCode `1110` 的系统扩展指令，通过 imm 低 3 位区分子操作：

| 指令 | 机器码 | 作用 |
|------|--------|------|
| **EI** | `E000` | Enable Interrupt，开中断 |
| **DI** | `E001` | Disable Interrupt，关中断 |
| **IRET** | `E002` | 从 ISR 返回，恢复 PC 与 IE |

译码：`id_stage.vhd`；提交：WB 级（`mem_wb_sys_ei/di/iret`）。

## 5. 关键信号（RTL / 波形）

### irq_timer / irq_pulse / ctrl_en（Timer 内部）

三者均在 `timer.vhd` 中，关系为 **`irq_timer <= irq_pulse`**。**`irq_timer` 即本课设的 Timer IRQ 线**（见 §2 **IRQ**）。

| 信号 | 类型 | 作用 |
|------|------|------|
| **`ctrl_en`** | 寄存器 | Timer **使能**：1=每拍 `counter` 减 1；0=停止倒计时。复位默认 1；可写 `0xFF20` bit0 控制 |
| **`irq_pulse`** | 内部单拍 | `counter` 减到 0 时置 1 **一个时钟**，表示发生一次溢出 |
| **`irq_timer`** | 输出端口 | 将 `irq_pulse` 送到 `interrupt_controller`，置位 `irq_pending` |

```text
ctrl_en=1 → counter: 80→79→…→0 → irq_pulse=1（单拍）→ irq_timer → pending←1
                              ↑ 溢出后 counter ← period_reg（不是 wdata）
```

### period_reg / counter / wdata（Timer 周期）

| 信号 | 作用 |
|------|------|
| **`period_reg`** | 保存**溢出周期**；复位为 `DEFAULT_PERIOD=80`；溢出后用来重载 `counter` |
| **`counter`** | **当前倒计时**；每拍减 1，到 0 产生 `irq_pulse` |
| **`wdata`** | 仅当 CPU **`ST` 写 `0xFF24`** 时，用写入值更新 `period_reg` 和 `counter`（软件改周期） |

**常见误解**：倒计时默认就用 80（存在 `period_reg` 里），**不是**每拍读 `wdata`。`wdata` 只是 MMIO 写周期寄存器时的配置数据。

### ADDR_CTRL / ADDR_PERIOD（Timer 地址常量）

`timer.vhd` 中用于**地址译码**，区分 CPU 访问的是哪个 MMIO 寄存器：

| 常量 | 值 | 寄存器 |
|------|-----|--------|
| `ADDR_CTRL` | `0xFF20` | 控制寄存器（按位定义，见下） |
| `ADDR_PERIOD` | `0xFF24` | 周期寄存器（`wdata` 整 16 位为周期） |

> **本课设规定**（`timer.vhd`）：**`0xFF20(0)`** = Timer 使能（`ctrl_en`）；**`0xFF20(1)`** = 写 1 时清脉冲并重载 `counter`。**`0xFF24` 无 bit0/bit1 含义**，写入的 16 位数即新周期。

**rdata 读回**（CPU `LD` 时）：读 **`0xFF20`** → bit0 = `ctrl_en`，其余位 = 0；读 **`0xFF24`** → 当前 **`period_reg`**（16 位）。中断不自动写 `rdata`。

### irq_pending / pending

中断请求**挂起标志**。Timer 触发后置 1，CPU 响应中断（`irq_ack`）后清 0。

即使 `IE=0`，pending 仍可置位，等 `EI` 后再响应。

### irq_ack

CPU 确认已接受中断，清除 pending。本项目 **`irq_ack <= irq_take`**。

### irq_take

组合逻辑：本拍**真正进入 ISR** 的条件：

```text
irq_take = irq_pending ∧ STATUS.IE ∧ wb_commit ∧ ¬cache_stall ∧ ¬(WB 为 IRET)
```

### iret_commit

WB 级 **IRET 指令提交**，触发 `PC ← EPC`、开中断、flush 流水线。

### wb_commit / mem_wb_valid

- **`mem_wb_valid`**：WB 级当前有真实指令（非 bubble）
- **`wb_commit`**：本拍为指令**提交边界**（且非 cache stall）

精确中断只在 `wb_commit=1` 时响应，保证已完成的指令不被打断。

### flush_all

中断进入或 IRET 返回时**冲刷流水线**的总控信号：

```text
flush_all = irq_take ∨ iret_commit
```

效果：IF/ID 插 NOP、ID/EX 插 bubble、EX/MEM 与 MEM/WB 清空有效位。

### pc_src / pc_redirect

IF 级 PC 来源选择（`if_stage.vhd`）：

| pc_src | 含义 |
|--------|------|
| `00` | PC+1 |
| `01` | 分支目标（BNE） |
| `10` | J 跳转目标 |
| `11` | `pc_redirect`（ISR 入口 `0x0100` 或 EPC） |

## 6. 中断模型

### 精确中断（Precise Interrupt）

在**指令边界**响应中断：WB 级指令完整提交后，再保存 EPC、flush 流水、跳转 ISR；不会“**半条指令**”被打断。

本项目通过 `wb_commit` + `mem_wb_valid` 实现。

### 关中断 / 开中断

| 操作 | 方式 | 时机 |
|------|------|------|
| 开中断 | `EI` 或 `IRET` | WB 提交 |
| 关中断 | `DI` 或硬件（`irq_take`） | WB 提交 / 进入 ISR |

进入 ISR 时自动关中断，防止 ISR 执行期间再次嵌套中断（单级模型）。

### MMIO 旁路（Bypass D-Cache）

访问 `0xFFxx` 时不经过 D-Cache，避免外设寄存器被 Cache 缓存导致读写错误。见 `soc_top` 信号 **`is_mmio`**。

## 7. 外设相关

### Timer（定时器）

MMIO 外设，详见 §5 **irq_timer / ctrl_en**、**period_reg / wdata** 及 **ADDR_CTRL / ADDR_PERIOD**。复位后默认 `ctrl_en=1`、周期 80 拍。

### UART（串口）

详见 §1 **UART**。MMIO 写 `0xFF00` 锁存发送数据；读 `0xFF04` 查询状态。

### GPIO（通用 I/O）

详见 §1 **GPIO**。MMIO 写 `0xFF10` 驱动 LED 寄存器。

---

## 8. 一句话串起来

> 在 **SoC** 中，**Timer** 溢出发出 **IRQ**（**irq_timer**）→ **中断控制器** 置 **irq_pending** → 主程序 **EI** 打开 **IE** 后，CPU 在 **WB 指令边界**（**wb_commit**）**响应中断**：保存 **EPC**、**关中断**、**flush** 流水线、PC 跳到 **ISR**（`0x0100`）；ISR 处理完后 **IRET** 从 **EPC** 恢复主程序。**外设**通过 **MMIO** 地址访问。
