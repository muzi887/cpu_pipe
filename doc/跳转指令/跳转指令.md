Hazard detect 单元位于 **ID 阶段**，跳转在 ID 判定（`Control.Jump`），跳转目标为 `IF/ID.NPC + uext(IF/ID.Ins[7:0])`。

---

## Hazard Unit 输出信号约定

| 信号 | 含义 |
|------|------|
| `Pc_en = 0` | 冻结 PC（stall） |
| `Ifid_en = 0` | 冻结 IF/ID |
| `Ifid_src = 0` | IF/ID ← `i_cache` 正常指令 |
| `Ifid_src = 1` | IF/ID ← `0`（NOP，冲刷误取指） |
| `Control_src = 0` | ID/EX ← bubble（控制全 0） |
| `Control_src = 1` | ID/EX ← Control 正常输出 |
| `Pc_src = 0` | PC ← PC+1 |
| `Pc_src = 1` | PC ← ID 级跳转目标（NPC + imm） |

**默认（无冒险、无跳转）：**

```text
Pc_en       = 1
Ifid_en     = 1
Ifid_src    = 0
Control_src = 1
Pc_src      = 0
```

---

## 一、无条件跳转 J（`Control.Jump = 1`）

J 在 **ID 阶段** 判定，IF 已多取 1 条顺序指令，只需 **冲刷 IF/ID 1 级**（不需 `Control_src = 0` 清 ID/EX）。

```text
IF (Control.Jump = 1)
   Pc_src   = 1          ; PC ← IF/ID.NPC + uext(IF/ID.Ins[7:0])
   Ifid_src = 1          ; 冲刷 IF 误取的顺序指令 → IF/ID ← 0
   Pc_en    = 1          ; 允许 PC 更新到新目标
   Ifid_en  = 1
```

---

## 二、条件跳转（`Jump = 1` 且依赖 `czvs` 标志）

图中 Hazard 单元有 `czvs` 输入，条件跳转在 **Jump 成立且标志满足** 时与无条件 J 做相同 flush；**不成立** 则顺序执行。

**以 JNZ（Z=0 跳转）为例：**

```text
IF (Control.Jump = 1  AND  czvs.Z = 0)
   Pc_src   = 1
   Ifid_src = 1
   Pc_en    = 1
   Ifid_en  = 1
```

**以 JZ（Z=1 跳转）为例：**

```text
IF (Control.Jump = 1  AND  czvs.Z = 1)
   Pc_src   = 1
   Ifid_src = 1
   Pc_en    = 1
   Ifid_en  = 1
```

**跳转不成立（条件不满足）：**

```text
IF (Control.Jump = 1  AND  czvs.Z = 1)     ; 以 JNZ 为例，Z=1 不跳
   Pc_src   = 0
   Ifid_src = 0
```

其他标志位写法相同，例如 JC：`Control.Jump = 1 AND czvs.C = 1`。

---

## 三、load-use stall（同 Hazard 单元，优先级高于跳转）

图中 `Mem_read`、`ID/EX.RD` 与 `IF/ID.RS/RD` 用于 load-use 检测；**stall 优先于跳转**：

```text
IF (ID/EX.Mem_read = 1  AND  (ID/EX.RD = IF/ID.RS  OR  ID/EX.RD = IF/ID.RD))
   Pc_en       = 0       ; 冻结 PC
   Ifid_en     = 0       ; 冻结 IF/ID
   Control_src = 0       ; 向 ID/EX 插入 bubble
```

---

## 四、与 BNE（EX 阶段分支）的区别
 Branch if Not Equal：两寄存器不等就按 相对偏移 跳转。
#### BNE 在 EX 判定

BNE 的语义是：

```
BNE rs1, rs2, offset ; if rs1 ≠ rs2 then PC ← PC + offset
```

要比较 `rs1` 和 `rs2`，必须：

1. ID：从寄存器堆读出两个操作数
2. EX：用比较逻辑判断 `rs1 ≠ rs2`

所以 `branch_taken` 最早在 EX 阶段 才能确定：

```
branch_taken = ID/EX.Branch AND (ID/EX.RS_val ≠ ID/EX.RD_val)
```

而 J 在 ID 看 `Control.Jump = 1` 就能跳转，不需要等 EX 运算，所以 J 只误取 1 条顺序指令；BNE 会多拖 1 个周期，误取 2 条。

#### BNE 需 flush **两级**（IF/ID + ID/EX）
Flush（冲刷） ：流水线里已经取进来、但不该执行的指令，强行作废，替换成 NOP（空操作），让这条指令流过后面各级时什么都不做。

```text
IF (ID/EX.Branch = 1  AND  branch_taken = 1)  
   Pc_src      = 2          ; PC ← branch_target（EX 级计算）
   Ifid_src    = 1          ; IF/ID ← 0
   Control_src = 0          ; ID/EX ← bubble
   Pc_en       = 1
   Ifid_en     = 1
```

ID 阶段（BNE 在 IF/ID 译码时）:
branch_target = IF/ID.NPC + sign_ext(IF/ID.offset)
锁入 ID/EX: { Branch, branch_target, RS_val, RD_val, ... }

EX 阶段（BNE 在 ID/EX 时）:

```
branch_taken = ID/EX.Branch AND (RS_val ≠ RD_val)
               ─────────────     ─────────────────
                 控制信号              一次比较
                 (AND 门)           (1 次 ALU SUB)
```

```
                    ┌── czvs ──> Z ──> NOT ──┐
			         │                                              │
ID/EX ──> Forward ───┼──> ALU ──> result ──> EX/MEM
  .Branch ──────────┼────────────────────────┼──> AND ──> branch_taken
  .branch_target ─────┼──────────────────────────────> Pc_src MUX
                    │
Forwarding unit（底部，已有）
```

（Branch 从 ID/EX 出来，和 czvs 的输出进 AND 门 → branch_taken → 反馈 PC / Hazard）

## 五、汇总表（跳转类）

| 场景        | 条件                                                     | 输出                                                        |
| --------- | ------------------------------------------------------ | --------------------------------------------------------- |
| 默认        | 无跳转、无 stall                                            | `Pc_src=0, Ifid_src=0, Control_src=1, Pc_en=1, Ifid_en=1` |
| 无条件 J     | `Control.Jump = 1`                                     | `Pc_src=1, Ifid_src=1, Control_src=1, Pc_en=1, Ifid_en=1` |
| 条件 J（JNZ） | `Jump=1 AND Z=0`                                       | `Pc_src=1, Ifid_src=1, Control_src=1, Pc_en=1, Ifid_en=1` |
| 条件 J 不成立  | `Jump=1 AND Z=1`（JNZ）                                  | `Pc_src=0, Ifid_src=0, Control_src=1, Pc_en=1, Ifid_en=1` |
| BNE 成立    | `branch_taken=1`（`Branch AND NOT Z`，EX 级 ALU 做 SUB 后判） | `Pc_src=2, Ifid_src=1, Control_src=0, Pc_en=1, Ifid_en=1` |
| BNE 不成立   | `Branch=1 AND Z=1`（RS=RD，不跳）                           | `Pc_src=0, Ifid_src=0, Control_src=1, Pc_en=1, Ifid_en=1` |
| load-use  | `Mem_read=1 AND RD 命中 RS/RD`                           | `Pc_en=0, Ifid_en=0, Control_src=0`                       |

## 时序示意（无条件 J）

```text
周期 T:   IF 取 PC+1（误取）    ID 译码 J（Jump=1）→ Pc_src=1, Ifid_src=1
周期 T+1: IF 从新 target 取指   IF/ID ← 0（NOP，误取被冲掉）
```

J 在 ID 判定只需 **1 条 flush**；BNE 在 EX 判定需要 **2 条 flush**（IF/ID + ID/EX）。若你实际用的是 J 型 12 位绝对地址而非 `NPC + Ins[7:0]`，把 `Pc_src=1` 的数据源改成绝对 `target` 即可，Hazard 条件逻辑不变。


## Flush 和 Stall 的区别

|       | Stall（停顿）          | Flush（冲刷）            |
| ----- | ------------------ | -------------------- |
| 场景    | load-use 数据冒险      | 分支/跳转控制冒险            |
| PC    | 冻结不动（`Pc_en=0`）    | 更新到新地址（`Pc_en=1`）    |
| IF/ID | 保持原内容（`Ifid_en=0`） | 写入 NOP（`Ifid_src=1`） |
| 效果    | 同一指令再等 1 拍         | 错指令被扔掉，换成空操作         |
| 典型例子  | `LD` 后下一条立刻用结果     | `BNE` 成立跳走           |

Stall: “先别往前走，等数据准备好”
Flush: “继续走，但把错指令换成 NOP 扔掉”
