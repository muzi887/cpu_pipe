# 数据通路修改建议 + 斐波那契汇编

对照你当前图（IF/ID → ID 读寄存器 → 数据直通 → WB 后 MUX）和课设文档，要跑斐波那契需要补全 **EX/MEM 真实功能** 和 **冒险处理**。下面按优先级给出修改建议，并给出与当前 **16 位格式** 匹配的汇编与机器码。

---

## 一、当前图的主要问题

| 问题 | 现状 | 斐波那契需要什么 |
|------|------|------------------|
| **无 ALU** | RS/RD/imm 直通 | `ADD`、`ADDI`、`ptr++`、`cnt--` |
| **无数据存储器** | MEM 空过 | `ST` 写 `Mem[13..17]` |
| **WB MUX 选 RD/Rd_src** | 写回的是寄存器原值 | 应写 **ALU 结果** 或 **Load 数据** |
| **Wr_addr 接 Ins[7:4]** | 写地址是当前 IF 指令的 rd | 应写 **MEM/WB 里锁存的 rd**（否则写错寄存器） |
| **无分支逻辑** | PC 只能 +1 | `BNE` 循环 |
| **无转发/stall** | — | `ADD` 后 `ADDI x2,x3,0` 会 RAW |
| **IF/ID 缺 pc** | — | 分支目标 `PC + offset` 需要 |

---

## 二、数据通路修改建议（分 3 步）

### 第 1 步：补 EX + 改 WB（最小可算）

在 **ID/EX 与 EX/MEM 之间** 加入：

```text
Forward MUX A ──> ALU(A)  
Forward MUX B ──> ALUSrc MUX ──> ALU(B)   ; 0=RD/reg  1=imm
                      ↑
                 ID/EX.imm (16-bit extend)

ALU.result ──> EX/MEM.alu_result
```

ALU 的第一个输入（A 端）前面，有一个“转发多路选择器”

**WB 修改：**

```text
删除：WB 后 “RD vs Rd_src” 的旧 MUX

改为：
  MEM/WB.alu_result ──> writedata ──> Registers   ; 当前只有算术/立即数

Wr_addr 改为：
  ID 阶段 Ins[7:4] ──锁存──> ID/EX.rd ──> EX/MEM.rd ──> MEM/WB.rd ──> Wr_addr
  （绝不能 WB 时直接用当前 Ins[7:4]）
```

**ID/EX 需锁存：**

```text
{ RS, RD, imm, rd, control(EX段) }
```

此步可跑：`ADDI`、`ADD`、寄存器写回，**还不能**访存和分支。

---

### 第 2 步：补 MEM + MemToReg（可 Store）

在 **EX/MEM 与 MEM/WB 之间** 加入 **d_mem**：

```text
EX/MEM.alu_result ──> Addr
EX/MEM.RD 或 rs2_val ──> WriteData   ; ST 用
MemRead / MemWrite ──> d_mem
d_mem.ReadData ──> MEM/WB.mem_data
```

**WB 前加 MemToReg MUX：**

```text
MemToReg MUX:
  0 ── alu_result   (ADD / ADDI)
  1 ── mem_data     (LD，以后扩展)
        │
        └──> writedata
```

此步可跑：`ST x3, 0(x4)` 写斐波那契结果到主存。

---

### 第 3 步：补分支 + 冒险单元（完整流水）

**EX 阶段增加：**

```text
branch_taken = Branch && (RS != RD)    ; BNE
branch_target = IF/ID.pc + sign_extend(offset)

PC MUX:
  0: pc_plus1
  1: branch_target
  2: jump_target (可选)
```

**IF/ID 需增加：** `pc`（取指时的 PC，供分支计算）

**hazard_unit + forwarding_unit：**

```text
转发：
  EX/MEM.alu_result ──> Forward MUX
  MEM/WB.write_data ──> Forward MUX

Stall（load-use，若以后加 LD）：
  PC_hold, IF/ID_hold, ID/EX_flush

Flush（BNE 成立）：
  IF/ID <= NOP, ID/EX <= NOP, PC <= branch_target
```

---

### 修改后总体结构

```text
IF:   PC, PC+1, i_cache/instr_mem ──> IF/ID {pc, pc+1, instr}

ID:   Control, Registers(RS,Rd), imm_extend
      ──> ID/EX {RS, RD, imm, rd, pc, control}

EX:   Forward MUX + ALU + Branch
      ──> EX/MEM {alu_result, rs2_val, rd, branch_taken, branch_target, control}

MEM:  d_mem (Addr=alu_result)
      ──> MEM/WB {mem_data, alu_result, rd, control}

WB:   MemToReg MUX ──> writedata
      MEM/WB.rd ──> Wr_addr,  Reg_w ──> Registers
```

**i_cache**：第一版可先用 **instr_mem 直连 main_memory**（文档 P0 不要求 I-Cache）。

## 三、指令编码（匹配你当前 16 位格式）

沿用你的字段划分：

```text
| Ins[15:8] opcode | Ins[7:4] | Ins[3:0] |
|     8 bit        |  4 bit   |  4 bit   |
```

| 指令 | opcode | [7:4] | [3:0] | 语义 |
|------|--------|-------|-------|------|
| **ADDI** | `0x01` | rd | imm | rd ← 0 + imm（初始化） |
| **ADDI2** | `0x11` | rd | rs | rd ← rs + imm，imm 在 opcode 低 4 位 `[11:8]` |
| **ADD** | `0x2d` | rs2 | rs1 | rd←opcode低4位，rd ← rs1 + rs2 |
| **ST** | `0x30` | base | data | Mem[reg[base]] ← reg[data] |
| **BNEZ** | `0x50` | offset₄ | rs | if reg[rs]≠0 then PC ← PC+1+sign_ext(offset) |
| **HALT** | `0xFF` | 0 | 0 | 停止 |

说明：

- **ADD** 的 rd 放在 `opcode[3:0]`，例如 ADD x3,x1,x2 → `0x23` + `[7:4]=x2` + `[3:0]=x1` → **`0x2321`**
- **ADDI2** 的 imm 在 `[11:8]`，例如 ADDI x1,x2,0 → **`0x1102`**（imm=0, rd=1, rs=2）
- **BNEZ** offset 为 **4 位有符号**，回跳 -6 → `1010₂` = `0xA`

## 四、寄存器约定（与课设文档一致）

```text
x0 = 0        （硬件恒 0）
x1 = a        当前 f[i]
x2 = b        当前 f[i+1]
x3 = tmp      新项
x4 = ptr      写地址，从 13 起
x5 = cnt      循环次数 n-2 = 5
```

## 五、斐波那契汇编程序

计算第 7 项：`Mem[11]=1, Mem[12]=1`，循环 5 次，结果 `Mem[13..17] = 2,3,5,8,13`，最终 `x3=13`。

```asm
; ========== 初始化 ==========
        ADDI    x1, 1           ; a = 1
        ADDI    x2, 1           ; b = 1
        ADDI    x4, 13          ; ptr = 13
        ADDI    x5, 5           ; cnt = 5  (n-2)

; ========== 循环 ==========
loop:   ADD     x3, x1, x2      ; tmp = a + b
        ST      x3, x4          ; Mem[ptr] = tmp
        ADDI2   x1, x2, 0       ; a = b
        ADDI2   x2, x3, 0       ; b = tmp
        ADDI2   x4, x4, 1       ; ptr++
        ADDI2   x5, x5, -1      ; cnt--  (imm=0xF = -1)
        BNEZ    x5, -6          ; if cnt!=0 goto loop
        HALT
```

## 六、机器码对照表

| 地址 | 汇编 | 机器码 (hex) | 编码说明 |
|------|------|--------------|----------|
| 0x00 | ADDI x1, 1 | `0111` | rd=1, imm=1 |
| 0x01 | ADDI x2, 1 | `0121` | rd=2, imm=1 |
| 0x02 | ADDI x4, 13 | `014D` | rd=4, imm=13 |
| 0x03 | ADDI x5, 5 | `0155` | rd=5, imm=5 |
| 0x04 | ADD x3, x1, x2 | `2321` | op=0x23, rs2=2, rs1=1 |
| 0x05 | ST x3, x4 | `3043` | base=4, data=3 |
| 0x06 | ADDI2 x1, x2, 0 | `1102` | imm=0, rd=1, rs=2 |
| 0x07 | ADDI2 x2, x3, 0 | `1203` | imm=0, rd=2, rs=3 |
| 0x08 | ADDI2 x4, x4, 1 | `1144` | imm=1, rd=4, rs=4 |
| 0x09 | ADDI2 x5, x5, -1 | `11F5` | imm=0xF, rd=5, rs=5 |
| 0x0A | BNEZ x5, -6 | `50A5` | offset=0xA(-6), rs=5 |
| 0x0B | HALT | `FF00` | |

**BNEZ 回跳验证：** `BNEZ` 在地址 `0x0A`，`loop` 在 `0x04`，`offset = 0x04 - 0x0B = -7`？ 

PC+1 模型：执行 0x0A 时 PC 指向 0x0A，下一条顺序是 0x0B，要回到 0x04：offset = 0x04 - 0x0B = **-7**。

4 位有符号 -7 = `1001` = 0x9，不是 -6。

重新计算（分支在 EX，PC+1 为顺序下一条）：
- 若 `branch_target = pc + offset`，pc 为 BNEZ 的 IF/ID.pc = 0x0A
- 目标 0x04：offset = 0x04 - 0x0A = **-6** ✓ (0xA in 4-bit signed)

若用 `PC+1` 即 0x0B：offset = 0x04 - 0x0B = -7 = 0x9

课设文档写 `branch_target = ID/EX.pc + imm`，pc 是 BNE 指令自己的 pc。

假设 pc 锁存的是指令地址 0x0A，target=0x04，imm=-6 → **0x50A5** 正确（若 pc 是 0x0A）。

若 pc 锁存 PC+1 或取指地址不同，需仿真时对齐。常见：`pc = 当前指令地址`，offset = target - pc = -6。

## 九、本程序会触发的冒险（仿真可验证）

| 位置 | 类型 | 处理 |
|------|------|------|
| `ADD x3,x1,x2` → `ST x3,x4` | RAW | EX/MEM 转发 rs2 或直接用 x3 |
| `ADD x3,...` → `ADDI2 x2,x3,0` | RAW | **ForwardA/B**，EX/MEM → EX |
| `ADDI2 x5,x5,-1` → `BNEZ x5,-6` | RAW | MEM/WB 或 EX/MEM 转发 |
| `BNEZ` 跳转 | 控制冒险 | flush IF/ID、ID/EX 各 1 条 |

---

## 十、实施顺序建议

```text
① 加 ALU + 改 Wr_addr 流水线传递 + 删旧 MUX     → 跑 ADDI/ADD
② 加 d_mem + MemToReg MUX + ST                  → 能写 Mem[13..17]
③ 加 BNEZ + pc + flush                          → 能循环
④ 加 forwarding_unit + hazard_unit              → 结果正确、波形可验收 V3/V5
```
