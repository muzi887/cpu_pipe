

AX/BX/CX/DX 是**程序用的通用寄存器**，放在寄存器堆（reg_file）里
寄存器堆有 2 个读端口 + 1 个写端口

| 端口            | 类型  | 含义          | 注意              |
| ------------- | --- | ----------- | --------------- |
| **RS_addr**   | 读地址 | 要读哪个寄存器（编号） |                 |
| **RS**        | 读数据 | 该寄存器的值      | 输出              |
| **RD_addr**   | 读地址 | 第二个读端口的寄存器号 |                 |
| **RD**        | 读数据 | 第二个寄存器的值    | 输出              |
| **Wr_addr**   | 写地址 | 要写哪个寄存器     | 通常是目标寄存器 **rd** |
| **writedata** | 写数据 | 要写入的值       | 来自 WB           |
| **Reg_w**     | 写使能 | 是否写入        | 为 0 时不写         |

## 用汇编实现流水

```asm
; 初始化（对应前 4 条写 AX/BX/CX/DX）
ADDI x1, x0, 3      ; AX = 1+2 = 3
ADDI x2, x0, ...    ; BX = ...
ADDI x3, x0, 7      ; CX = 3+4 = 7
ADDI x4, x0, ...    ; DX = ...

; 后续累加（对应 AX+5, AX+7, CX+6 等）
ADDI x1, x1, 5      ; AX ← AX + 5
ADD  x1, x1, x2     ; AX ← AX + BX
ADDI x3, x3, 6      ; CX ← CX + 6
ADD  x3, x3, x4     ; CX ← CX + DX
; ...
```

硬件上每条指令都走同一套通路：

```text
ID：RS_addr/RD_addr 读 x1、x2 等
EX：ALU 运算
WB：Wr_addr 写回 x1、x3 等
```

**不需要**为 AX 单独做一个寄存器。

## 各部件分工（避免混淆）

```text
┌─────────────────────────────────────────┐
│  reg_file（寄存器堆）                     │
│  x0, x1(AX), x2(BX), x3(CX), x4(DX)... │  ← AX/BX/CX/DX 在这里
└─────────────────────────────────────────┘
         ↑ WB 写              ↓ ID 读

┌─────────────────────────────────────────┐
│  IF/ID, ID/EX, EX/MEM, MEM/WB           │  ← 流水线寄存器
│  存：instr, RS, RD, alu_result, rd...   │  ← 不存“AX 这个名字”
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  ALU                                    │  ← 图上“AX+BX”在 EX 算
└─────────────────────────────────────────┘
```

- **AX/BX/CX/DX**：reg_file 里的**编号/名字**
- **AX+BX**：EX 阶段 ALU 的**中间结果**（在 EX/MEM 里，WB 后才进 x1）

## 数据相关怎么处理

笔记中间涂阴影 = 有 RAW 相关，例如前一条写 AX，后一条马上 `AX+5`：

| 情况 | 处理 |
|------|------|
| 间隔足够（WB 后再读） | 直接从 reg_file 读，**不需转发** |
| 紧挨着（EX 就要用） | **转发** 或 **stall** |
| Load 后立刻用 | 必须 **stall 1 拍** |

AX/BX/CX/DX 仍在 reg_file；转发/stall 解决的是**读太早**，不是多加寄存器。
