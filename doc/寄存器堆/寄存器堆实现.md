# `ADDI rd, rs, imm` 如何用图中的寄存器堆实现

语义：**rd ← rs + imm**（把寄存器 `rs` 的值与立即数 `imm` 相加，结果写入 `rd`）。

下面按你图中的 **Registers** 端口，说明它在五级流水中如何工作。

---

## 1. 指令与寄存器堆端口的对应关系

你图中的 Registers：

```text
Reg_w      ← 写使能（来自 WB 控制信号）
RS_addr    ← Ins[3:0]   → 读出 RS
RD_addr    ← Ins[7:4]   → 读出 RD
Wr_addr    ← Ins[7:4]（经流水线传到 WB）
writedata  ← 运算结果（来自 WB）
```

对 `ADDI rd, rs, imm`，各字段作用：

| 字段 | 指令位 | 寄存器堆用法 |
|------|--------|--------------|
| **rs** | `[3:0]` | 接 **RS_addr**，**读**源操作数 → **RS** |
| **rd** | `[7:4]` | 不在 ID 读；锁入 ID/EX，WB 时作 **Wr_addr** |
| **imm** | 立即数字段 | **不经过寄存器堆**，走 **Unsigned extend** → ALU |

注意：你格式里 `[3:0]` 在“立即数指令”里也可能表示 imm。若 `ADDI rd, rs, imm` 中 **rs 和 imm 都要独立编码**，imm 往往放在 **opcode 的低几位** 或其它子字段；`[3:0]` 专门作 **rs 地址**。具体以你 opcode 表为准。

---

## 2. 五级流水逐步过程

### ID 阶段（图中 Registers 主要在这里工作）

```text
Ins[15:8] ──> Control
              ├─ ALUSrc = 1（ALU 第二输入选 imm）
              ├─ ALUOp  = ADD
              └─ Reg_w  = 1（WB 时写寄存器）

Ins[3:0]  ──> RS_addr ──> Registers ──> RS = reg[rs]
Ins[3:0]  ──> Unsigned extend ──> 16-bit imm
Ins[7:4]  ──> 锁入 ID/EX，作为 rd（以后 Wr_addr 用）

Ins[7:4]  ──> RD_addr ──> RD（ADDI 一般不用 RD 参与运算）
```

**读端口**：只用 **RS_addr** 读 `rs`；**RD_addr** 虽连着 `[7:4]`，ADDI 的 ALU 不用 RD，RD 读出可忽略。

**写端口**：ID 阶段 **不写**（`Reg_w = 0`）。

锁入 **ID/EX** 的内容：

```text
{ RS, imm, rd, control(ALUSrc=1, ALUOp=ADD, Reg_w=1, ...) }
```

---

### EX 阶段（Registers 不再参与，ALU 运算）

```text
ALU 输入 A ← RS（来自 ID/EX）
ALU 输入 B ← imm（ALUSrc=1，不选寄存器）
ALU 输出   ← RS + imm = alu_result
```

若有转发，Forward MUX 在 EX 选 RS；ADDI 通常不涉及转发。

结果锁入 **EX/MEM**：`{ alu_result, rd, control }`。

---

### MEM 阶段

ADDI **不访存**（MemRead=0, MemWrite=0），`alu_result` 直通到 **MEM/WB**。

---

### WB 阶段（Registers 写端口工作）

```text
Wr_addr    ← rd（从 ID/EX → EX/MEM → MEM/WB 一路传递）
writedata  ← alu_result
Reg_w      ← 1

→ Registers(rd) ← alu_result
```

三个写信号配合：

```text
Reg_w = 1  且  Wr_addr = rd  且  writedata = rs + imm
→ reg[rd] 被更新
```

---

## 3. 数据通路总览

```text
         ID 阶段                         EX 阶段              WB 阶段
         ───────                         ───────              ───────

Ins[3:0] ──> RS_addr ──> RS ──────────────────> ALU(A)
                                              ↗
Ins[?]   ──> extend ──> imm ──> ALUSrc ──────┘ ALU(B)
                                              │
                                              v
                                         alu_result
                                              │
Ins[7:4] ──> rd ──(流水线传递)───────────────┼──> Wr_addr
                                              │
                                              └──> writedata
                                                         │
Reg_w <──────────────────────────────────────────────────┘
         (WB 阶段 Reg_w=1 写入 Registers)
```

---

## 4. 具体例子

**`ADDI x3, x1, 5`**（x3 ← x1 + 5）

假设：x1=10，编码 `[7:4]=0011`(x3)，`[3:0]=0001`(x1)，imm=5 由扩展器给出。

| 阶段 | Registers 相关操作 |
|------|-------------------|
| **ID** | RS_addr=1 → RS=10；rd=3 锁入 ID/EX；Reg_w=0（还不写） |
| **EX** | ALU：10 + 5 = 15 |
| **MEM** | 不访存，15 继续传 |
| **WB** | Wr_addr=3，writedata=15，Reg_w=1 → **x3 ← 15** |

---

## 5. 三个端口在 ADDI 中的分工

| 端口 | ADDI 中是否使用 | 作用 |
|------|-----------------|------|
| **RS_addr → RS** | ✅ 使用 | 读 **rs** 作 ALU 第一操作数 |
| **RD_addr → RD** | ❌ 一般不用 | 虽连 `[7:4]`，ADDI 不用 RD 做运算 |
| **Wr_addr + writedata + Reg_w** | ✅ WB 使用 | 把 **rd ← rs + imm** 写回 |

---

## 6. 和你原模型机的对应

| 原微程序 | ADDI 在流水 CPU 中 |
|----------|-------------------|
| `Bus_Sel` 选 IR 立即数 | ID：imm 扩展 |
| `Bus_Sel` 选 BX/CX 等 | ID：RS_addr 读 rs |
| `LoadAC` + ALU + `AC_Sel←ALU` | EX：ALU 运算 |
| `LoadAX/LoadBX` | WB：Wr_addr + writedata + Reg_w |

---

## 一句话

**ADDI rd, rs, imm** 用图中寄存器堆的方式是：

1. **ID**：`RS_addr=rs` **读** RS；imm 走扩展器，**不读寄存器**；rd 暂存供 WB 用  
2. **EX**：ALU 算 **RS + imm**  
3. **WB**：**Reg_w=1**，**Wr_addr=rd**，**writedata=结果** → 完成 **rd ← rs + imm**

寄存器堆在 ADDI 里：**ID 读 rs，WB 写 rd**；imm 不经过 Registers。
